# 前端设计流程（NexusGate Admin）

## 1. 目标澄清
- 产品定位为运维管理台，而不是聊天前端。
- MVP 聚焦 6 个模块：Dashboard、上游配置、记忆中心、MemoryPack 预览、请求追踪、客户端接入。
- 核心验收：可看清当前生效配置、可追踪路由/trim/fallback/grounding、可浏览记忆结构。

## 2. 信息架构
- 一级导航固定为 6 个工作台页面。
- 每个页面遵循“概览指标 + 主操作区 + 证据/明细区”的布局。
- 所有关键决策数据（provider/model/fallback/unsupported_ratio）统一在右侧详情区可见。

## 3. 视觉系统（ui-ux-pro-max）
- 风格：运维工业风（深色石墨底 + 青绿色信息色 + 琥珀告警色）。
- 字体：`Space Grotesk`（标题）+ `IBM Plex Sans`（正文）。
- 组件：高对比卡片、分层阴影、清晰状态标签、可读表格。
- 动效：首屏渐入、卡片轻微上浮、导航切换淡入，避免过度动画。

## 4. 交互流程
- 导航切换采用单页工作台模式，保留上下文，降低认知切换成本。
- 配置页以“主配置(TARGET_*) + 兼容配置(LLMAPI_*)”并列展示，避免混淆。
- 追踪页强调请求链路：输入 -> 路由 -> fallback -> trim -> grounding -> 输出。

## 5. 响应式与可访问性
- 断点：`<=980px` 自动折叠为顶部导航。
- 按钮、输入、标签具备可见焦点样式。
- 语义色同时配合文字提示，不用纯颜色表达状态。

## 6. 交付策略
- 先交付可运行静态前端骨架（本次）。
- 下一步接入后端 admin API：`/admin/config`、`/admin/memories`、`/admin/traces` 等。
- 再补充真实数据源、保存操作、连接测试、导入导出。
